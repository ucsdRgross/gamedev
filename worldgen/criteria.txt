===================================================================================
FTL PROCEDURAL GENERATION ENGINE SPECIFICATION & CONSTRAINT RULES LEDGER
===================================================================================

1. LANDMASS BOUNDARY & CONTAINMENT RULES
   - The master world landmass must always form an organic island.
   - Land must NEVER breach, touch, or clip the square texture boundaries (768x768).
   - All steps (Tectonics, Ridges, Erosion) must respect a strict radial ocean mask falloff.

2. HYDROGRAPHY & EROSION SIMULATION RULES
   - Hydraulic erosion drops must spawn strictly on slopes to carve visible valleys.
   - Rivers must form long, continuous networks flowing downhill.
   - Lowland basins and tectonic rift trenches must pool water into visible lakes.
   - Rivers and lakes must be thickened to 2-3 pixels to stay visible when downsampled.

3. TECTONIC PLATE & FAULT LINE SCHEMATIC RULES
   - Plate seeds must use Poisson-style jittered grid distributions to prevent clustering.
   - Plate cells must be accurately classified as Land or Ocean based on local terrain heights.
   - Cell boundary interfaces (Fault Lines) must use domain-warped noise to avoid straight edges.
   - Diagnostic arrows, vectors, nodes, and keys must burn directly into the saved grid images.

4. CIVILIZATION & POISSON NODE RULES
   - City nodes must NEVER spawn on water tiles, river beds, or deep lake basins.
   - Cities must distribute across all available pieces of land (including large upper islands).
   - Island cities must only exist if their size is large enough to support gameplay.

5. GRAPH прогрессия & TRAVERSAL RULES
   - The start and end anchors must lock exclusively to the absolute largest contiguous landmass.
   - Start/End points must never spawn on isolated minor islands, rock clusters, or across water blocks.
   - Directional path arrows must point forward, showing the left-to-right progression.
   - Travel distances must use strict min/max limits to prevent messy zig-zag loops.
   - Isolated nodes or dead ends must be stripped via a Depth-First Search (DFS) pruning pass.

6. COMPOSITE IMAGE LAYOUT CONFIGURATION MATRIX
   - The master overview grid map must use a strict 3x3 matrix layout (9 distinct quadrants).
   - Array snapshots must execute and register in precise sequential layout order:
	 [Row 0, Col 0] -> Snapshot Index 0: Landmass Base Template
	 [Row 0, Col 1] -> Snapshot Index 1: Tectonics Height Modifications
	 [Row 0, Col 2] -> Snapshot Index 2: Tectonics Diagnostic Vectors & Fault boundaries
	 [Row 1, Col 0] -> Snapshot Index 3: Peaks and Valleys Fine Ridges
	 [Row 1, Col 1] -> Snapshot Index 4: Hydraulic Erosion Softening Channels
	 [Row 1, Col 2] -> Snapshot Index 5: High-Contrast Pure Rivers & Lakes Overlay
	 [Row 2, Col 0] -> Snapshot Index 6: Multi-Hazard Biome Climate Map Layout
	 [Row 2, Col 1] -> Snapshot Index 7: Poisson Disc Sampled Civilizations
	 [Row 2, Col 2] -> Snapshot Index 8: Pruned Forward Traversal Graph Network
